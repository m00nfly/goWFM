为什么你需要 goWFM？

如果这些小麻烦你也遇到过，goWFM 也许正合适

大量文件需要远程访问

服务器、NAS、树莓派上有大量文件，想在任何设备上随时安全访问，不再局限于本地终端。

不想迁移，不想动目录

文件已有清晰的目录结构，不想迁移到第三方网盘，也不想被迫改变文件组织方式。

简易方案缺乏安全性

Nginx autoindex、Python http.server 虽然能用，但没有用户认证、没有权限控制，裸奔在公网太危险。

分享文件不想给全部权限

需要把几个文件发给别人，又不想重新上传到公共网盘，更不想开放整个目录。

核心特性

从最初的目录浏览，到现在该有的小工具基本都配齐了

多用户与权限

管理员账号可自定义，浏览、下载、全局上传、分享和日志权限可以分别分配。

保留原始目录结构

文件在磁盘上什么样，Web 上看到就什么样。不迁移、不拷贝，直接管理你的现有文件。

多文件分享

一个链接可以包含多个文件，支持名称、有效期、下载统计、短 ID 和一次性下载链接。

跨目录批量操作

切换目录时保留已选文件，可一次完成批量下载、分享或删除。

TOTP 与登录防护

支持验证码、登录封锁、TOTP、恢复码、受信任设备和管理员强制策略。

邮件通知与密码找回

SMTP 测试通过后，可发送分享通知和一次性密码重置邮件。

管理员仪表盘

存储空间、文件类型、用户、分享、操作趋势和安全检查都能一页查看。

完整操作日志

登录、文件、分享和配置修改都能筛选查看，匿名访问也会记录。

所有权与个人头像

记录文件创建者，管理员可调整所有者，用户也能设置头像和个人信息。

明暗主题与品牌设置

自定义站点名称、Logo、主题色、登录背景和 Markdown/HTML 品牌面板。

响应式界面

桌面、平板和手机都能用，文件列表、分享页面和设置页面会自动适配。

单文件运行

Go + SQLite，前端资源直接嵌入程序，不需要额外安装数据库服务。

使用场景

个人服务器、NAS 和几个人的小团队都能用

小团队共享文件

给每个人分配合适的权限，构建产物、文档和素材放在一个地方。

给客户发送大文件

把多个文件放进一个限时链接，对方无需注册就能在浏览器里下载。

个人 NAS 远程访问

在家里的 NAS 或树莓派上运行 goWFM，出门在外也能随时访问你的文件。

临时收集上传材料

给参与者创建上传账号，谁传了什么都能在所有权和日志里查到。
