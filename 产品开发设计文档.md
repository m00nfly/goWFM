## 1. 产品概述

### 1.1 项目背景

设计一套轻量级、自包含的Web文件管理系统，可部署为单一二进制文件，适用于个人、小团队或内网环境下的文件集中存储与管理。系统强调开箱即用、权限精细控制、操作可追溯。

### 1.2 核心目标

- 提供基于Web的本地文件浏览、上传、下载、分享能力。
- 支持多用户管理及细粒度权限分配。
- 完整的操作日志审计功能。
- 前后端集成于单一可执行文件，简化部署。
- 界面美观、现代、响应式，适配PC/平板/手机。

### 1.3 关键特性摘要

| 特性 | 描述 |
| ------ | ------ |
| 文件管理 | 浏览、上传、下载、删除、创建目录 |
| 分享链接 | 生成有时效性的公开下载链接 |
| 多用户 | 内置admin，可创建普通用户并分配权限 |
| 权限模型 | 基于RBAC，支持浏览、下载、上传、分享、日志管理 |
| 所有权管理 | 文件/目录与创建者绑定，admin可变更所有权 |
| 日志审计 | 记录所有关键操作，支持多维度筛选 |
| 部署形态 | Golang后端嵌入Vue前端静态资源，SQLite嵌入式数据库，单文件发布 |

## 3. 功能规格详细说明

### 3.1 配置文件 (`config.json`)

启动时从工作目录读取 `config.json`，结构如下：

```json
{
  "org_name": "some org name",
  "org_link": "https://somelinks.com",
  "data_root_path": "/absolute/path/to/data",
  "server_port": 8080,
  "session_secret": "auto-generated-or-user-defined",
  "log_level": "info"
}
```

- `org_name`: 选填，设置公司组织名称，用于UI展示
- `org_link` : 选填，设置 org_name 的超链接地址，用于UI 页脚版权信息显示
- `data_root_path`: 必填，所有文件操作均限制在此目录内（防止路径穿越）。
- `server_port`: 默认 8080。
- `session_secret`: 用于加密Session，若为空则程序随机生成。
- `log_level`: debug/info/warn/error。

### 3.2 首次运行向导

- 首次启动检测数据库中是否存在 `admin` 用户。
- 若无，前端首次访问任意页面时重定向至 `/setup` 初始化页面。
- `/setup` 页面引导用户设定 `config.json` 中的所有配置选项，设置 admin 密码（用户名固定为 `admin`）。
- 提交后更新 `config.json` 配置文件内容，并初始化数据库及创建 admin 用户，赋予所有权限，跳转至登录页。

### 3.3 用户认证与会话管理

- 采用 Session-Cookie 机制。
- 登录接口 `/api/auth/login`，成功后返回 HttpOnly Cookie。
- 会话超时时间默认 7 天，可配置。
- 登出接口 `/api/auth/logout`。

### 3.4 多用户管理与权限

#### 3.4.1 用户属性

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | integer | 主键 |
| username | text | 唯一，登录账号 |
| password_hash | text | bcrypt 哈希 |
| display_name | text | 显示名称（可修改） |
| email | text | 邮箱（可修改） |
| is_admin | bool | 是否为超级管理员（保留字段，与权限位独立） |
| permissions | integer | 位掩码权限，见下 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

#### 3.4.2 权限位定义

| 权限名 | 位值 | 说明 |
| -------- | ------ | ------ |
| PERM_BROWSE | 1 << 0 (1) | 浏览文件列表 |
| PERM_DOWNLOAD | 1 << 1 (2) | 下载文件 |
| PERM_UPLOAD | 1 << 2 (4) | 上传文件、创建目录 |
| PERM_SHARE | 1 << 3 (8) | 生成/管理分享链接 |
| PERM_MANAGE_LOGS | 1 << 4 (16) | 查看/筛选操作日志 |

- `admin` 账户 `permissions` 字段置为所有位的组合（例如 31）。
- `is_admin` 标记为 true 时，前端可展示“所有权管理”入口，且后端跳过所有权检查。

#### 3.4.3 用户管理接口（admin 可见）

- 列出用户 `GET /api/users`
- 创建用户 `POST /api/users`
- 更新用户权限/信息 `PUT /api/users/:id`
- 删除用户 `DELETE /api/users/:id`

### 3.5 文件/目录所有权模型

- 数据库表 `file_metadata` 记录通过 Web 创建的文件/目录信息：

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | integer | 主键 |
| file_path | text | 相对于 data_root_path 的路径（唯一索引） |
| is_directory | bool | 是否为目录 |
| owner_id | integer | 创建者用户ID（外键 users.id） |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

- **非 Web 创建的文件/目录**：数据库无记录，前端展示时所有者显示为 `admin`，创建时间显示为文件系统创建时间。
- **删除权限规则**：
  - 普通用户只能删除自己创建的文件/目录。
  - admin 可以删除任何文件/目录。
  - 目录非空时不允许删除（前端提示并阻止）。

### 3.6 文件浏览与列表展示

#### 3.6.1 接口设计

`GET /api/files?path={relativePath}`

- `relativePath` 为空或 "/" 表示根目录。
- 返回当前目录下的所有条目列表，包含：
  - 名称、大小、修改时间、是否为目录
  - 所有者信息（用户名）
  - 创建时间（优先使用数据库记录）
  - 当前用户对该条目的操作权限（是否可删除、下载、分享等）

#### 3.6.2 前端展示要求

- 面包屑导航显示从“根”开始的相对路径。
- 文件列表以表格或卡片形式展示（移动端自适应为列表）。
- 显示列：名称、大小、修改时间、所有者。
- 操作按钮根据权限动态渲染：
  - 删除按钮：仅当可删除时显示。
  - 下载按钮：仅当有下载权限时显示（文件）。
  - 分享按钮：仅当有分享权限时显示（文件）。
- 上传区域：有上传权限时显示“上传文件”和“新建文件夹”按钮。

### 3.7 文件上传与创建目录

#### 3.7.1 上传流程

1. 用户点击“上传文件”，弹出模态框。
2. 模态框内：
   - 显示当前目录路径（可编辑输入框，支持选择子目录）。
   - 文件选择器（单选，仅单文件上传）。
3. 提交后，前端逐文件或打包上传。
4. 后端处理：
   - 验证目标路径仍在 `data_root_path` 内。
   - 保存文件至磁盘。
   - 在 `file_metadata` 表中插入记录（owner=当前用户，创建时间=当前时间）。
5. 上传成功后刷新列表。

#### 3.7.2 创建目录

- 类似上传，提供输入框指定目录名。
- 后端创建目录并插入元数据记录。

### 3.8 文件下载

- 接口 `GET /api/download?path={relativeFilePath}`
- 验证用户下载权限。
- 返回文件流，Content-Disposition 设置为 attachment。

### 3.9 文件分享功能

#### 3.9.1 创建分享链接

- 接口 `POST /api/shares`
- 请求体：

  ```json
  {
    "file_path": "相对路径",
    "expire_days": 3
  }
  ```

- 后端生成唯一 token (UUID)，记录到 `shares` 表：

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | integer | 主键 |
| token | text | 唯一标识，URL 使用 |
| file_path | text | 相对路径 |
| owner_id | integer | 创建分享的用户 |
| expire_at | datetime | 过期时间（NULL表示永久） |
| created_at | datetime | 创建时间 |
| access_count | integer | 访问次数 |

- 返回分享链接：`http(s)://domain/share/{token}`

#### 3.9.2 访问分享链接

- `GET /share/{token}`
- 校验 token 有效性、文件存在性、是否过期。
- 若有效，增加访问计数，返回文件下载流（无需登录）。

#### 3.9.3 “我的分享”管理

- `GET /api/shares/my` 列出当前用户创建的所有分享。
- `DELETE /api/shares/:id` 删除分享链接（使其失效）。

### 3.10 操作日志功能

#### 3.10.1 日志记录表

`operation_logs`

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | integer | 主键 |
| user_id | integer | 操作者ID |
| action | text | 操作类型：LOGIN, CREATE_DIR, UPLOAD, DOWNLOAD, DELETE_FILE, DELETE_DIR, SHARE_CREATE, SHARE_ACCESS, CHANGE_OWNER, ... |
| target_path | text | 目标文件/目录相对路径 |
| details | text | JSON 格式附加信息 |
| ip_address | text | 客户端IP |
| created_at | datetime | 操作时间 |

#### 3.10.2 日志筛选接口

`GET /api/logs?start_time=&end_time=&user_id=&action=&target_path=`

仅拥有 `MANAGE_LOGS` 权限者可访问。

#### 3.10.3 前端日志查看器

- 表格展示，支持按时间、用户、操作类型、目标路径筛选。
- 支持分页。

### 3.11 所有权管理（admin 专属）

- 前端文件列表中，所有者列可点击（仅 admin 可见交互）。
- 点击后弹出下拉菜单，列出所有用户账号，选择后调用 `PUT /api/files/owner` 更新所有权。
- 接口更新 `file_metadata.owner_id` 字段。

### 3.12 用户自服务（账户设置）

- 右上角头像/图标点击展开菜单，包含“个人设置”、“退出登录”。
- 个人设置页面可修改：
  - 显示名称
  - 邮箱地址
  - 密码（需验证当前密码）
- 接口 `PUT /api/users/me`

## 4. 数据库设计详表

### 4.1 `users` 表

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    display_name TEXT DEFAULT '',
    email TEXT DEFAULT '',
    is_admin BOOLEAN DEFAULT 0,
    permissions INTEGER DEFAULT 1, -- 默认只有浏览权限
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 4.2 `file_metadata` 表

```sql
CREATE TABLE file_metadata (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    file_path TEXT UNIQUE NOT NULL,
    is_directory BOOLEAN DEFAULT 0,
    owner_id INTEGER NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (owner_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### 4.3 `shares` 表

```sql
CREATE TABLE shares (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    token TEXT UNIQUE NOT NULL,
    file_path TEXT NOT NULL,
    owner_id INTEGER NOT NULL,
    expire_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    access_count INTEGER DEFAULT 0,
    FOREIGN KEY (owner_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### 4.4 `operation_logs` 表

```sql
CREATE TABLE operation_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    action TEXT NOT NULL,
    target_path TEXT,
    details TEXT,
    ip_address TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### 4.5 `sessions` 表（可选，或用内存存储）

若需要持久化会话（支持服务重启不丢失登录），可建表：

```sql
CREATE TABLE sessions (
    token TEXT PRIMARY KEY,
    user_id INTEGER NOT NULL,
    expires_at DATETIME NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

## 7. 安全规范与边界条件

### 7.1 路径安全

- 所有文件操作必须基于 `data_root_path` 拼接并解析为绝对路径后，验证路径前缀是否等于 `data_root_path`。
- 禁止包含 `..` 或符号链接跳出根目录。

### 7.2 文件上传安全

- 限制上传文件大小（可配置，默认 1GB）。
- 文件名过滤非法字符，推荐保留字母数字、点、下划线、中文。
- 重名处理策略：自动重命名为 `filename (1).ext` 样式。

### 7.3 删除保护

- 删除目录前检查目录是否为空（通过系统调用读取目录内容）。
- 软删除不实现，直接物理删除。

### 7.4 登录防护

- 登录失败延迟响应（防暴力破解）。
- 密码使用 bcrypt 存储。

### 7.5 跨站防护

- Cookie 设置 `HttpOnly`, `SameSite=Strict`（或Lax）。
- 所有状态变更 API 建议附加 CSRF Token（或使用 `SameSite=Strict` 缓解）。

## 9. 构建与打包说明

### 9.1 后端嵌入前端资源

使用 Go 1.16+ `embed` 特性：

```go
//go:embed frontend/dist/*
var staticFiles embed.FS
```

将 `/` 及 `/assets/*` 等路径映射到内嵌文件系统。

### 9.2 编译命令

```bash
cd frontend && npm run build
cd ../backend
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -o gowfm .
```

生成无依赖的二进制文件 `gowfm`。

### 9.3 启动流程

1. 读取 `config.json`（若不存在则退出并提示创建）。
2. 初始化 SQLite 数据库连接，自动迁移表结构。
3. 启动 HTTP 服务。
