# Go Web File Manager (goWFM)

**简单 · 轻量 · 易用**  
一键让本地文件夹秒变安全私有网盘，无需改变原有目录结构。

---

## ✨ 为什么你需要 goWFM？

- 你有一台服务器/NAS/树莓派，存着大量文件，想随时随地通过浏览器访问。
- 你不想把文件迁移到第三方网盘，**原有目录结构一个字母都不想动**。
- 你试过用 Nginx/Apache 的 `autoindex`，但那是“裸奔”——没有登录、没有权限、没有分享控制。
- 你需要给同事/家人分享某个文件，但又不想给他们整台服务器的访问权。

**goWFM 就是为此而设计：** 它像传统的 HTTP 文件服务器一样直接暴露本地目录，却拥有现代网盘的**用户体系、权限控制、分享链接、操作审计**能力。一个二进制文件，丢到服务器上就能跑。

## 📦 特性一览

- 🔐 **多用户与权限**  
  内置 admin 超级管理员；可创建多个普通账户，按需分配浏览、下载、上传、分享、查看日志等权限。

- 📁 **保留原始目录结构**  
  直接基于你指定的本地文件夹（`data_root_path`），文件在磁盘上是什么样，Web 上看到的就是什么样。原有文件自动归属 admin，无需任何迁移。

- 🔗 **文件分享链接**  
  点击即可生成有时效性的公开下载链接（支持设定有效期），无需登录即可访问，完美替代临时文件传输场景。

- 🧾 **操作日志审计**  
  谁在什么时候上传/删除了什么文件？所有操作均有迹可循，支持按时间、用户、动作、目标文件筛选。

- 🧑‍💼 **所有权管理**  
  每个文件/文件夹都记录创建者，用户只能删除自己创建的内容。admin 可以变更 文件/文件夹的“所有者”。

- 📱 **现代响应式界面**  
  基于 Vue 3 + TypeScript 打造，UI 清爽美观，完美适配电脑、平板、手机。

- 🪶 **极致轻量**  
  后端使用 Go 编写，数据库为嵌入式 SQLite，前端打包后全部嵌入二进制。编译后仅一个可执行文件，无任何外部依赖。

- 🚪 **开箱即用**  
  下载 → 运行 → 浏览器打开 → 完成初始化设定 即可使用。

## ⚡ 快速开始

### 1️⃣ 下载gowfm二进制文件

从本仓库的 `release` 页面下载对应操作系统平台最新的 `gowfm` 文件到本地

### 2️⃣ 运行

```bash
# 赋予执行权限（Linux/macOS）
chmod +x gowfm

# 直接运行
./gowfm
```

### 3️⃣ 访问与初始化

浏览器打开 `http://your-server-ip:8080`  
首次访问会跳转到初始化页面，设置 `admin` 账户密码与设定要共享的本地目录路径，之后就可以登录使用了。

### 4️⃣ （可选）创建其它用户

以 admin 身份登录后，进入“用户管理”可创建其他账户，并分配相应权限。

## 🖥️ 从源码构建

### 前置要求
- Go 1.22+
- Node.js 20+ 与 npm

### 构建步骤

```bash
# 1. 拉取源代码
git clone https://github.com/m00nfly/goWFM.git

# 2. 编译构建
make

# 得到的 gowfm 二进制文件即可分发部署
```

> **跨平台编译**：  
> `make build-linux`  
> `make build-windows`
> `make build-darwin`

## 🧭 使用场景举例

| 场景 | 传统工具痛点 | goWFM 方案 |
|------|--------------|--------------|
| **团队共享项目文档** | 用 Samba/FTP 需要每个成员设置账号，权限粗糙。 | 管理员创建只读/读写账号，每个人只能看到和操作自己被授权的部分。 |
| **给客户发送大文件** | 邮箱附件限制，网盘需要对方注册。 | 生成一个带有效期分享链接，客户直接浏览器下载。 |
| **个人 NAS 文件远程访问** | 自建 Nginx autoindex 缺乏安全性，暴露所有文件。 | goWFM 提供登录保护，并可精确控制每个子账户的访问范围。 |
| **临时收集多人上传的材料** | 开一个公共 FTP 或网盘收集链接，无法追溯谁传了什么。 | 为每个人创建上传账号，操作日志清晰记录每份文件的来源。 |

## 🔧 配置详解

`config.json` 完整字段说明（可通过初始化页面进行设定）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `data_root_path` | string | Yes | 文件系统根路径，所有文件操作均限制在此目录内。 |
| `server_port` | int | No | 监听端口，默认 `8080`。 |
| `session_secret` | string | No | 用于加密 Session 的密钥。留空则每次启动随机生成（重启后所有用户需重新登录）。生产环境建议固定一个强随机字符串。 |
| `log_level` | string | No | 日志级别，可选 `debug`、`info`、`warn`、`error`，默认 `info`。 |
| `db_path` | string | No | matedata数据路路径，默认 `gowfm.db`。|
| `max_upload_size` | int | No | 允许上传文件的最大大小，单位 b，默认值 0 （1GB）|

## 🧱 技术栈

| 端 | 技术 |
|----|------|
| 后端 | Go 1.22+，纯标准库 HTTP（无框架依赖），SQLite（modernc.org/sqlite，纯 Go 实现） |

| 嵌入 | Go `embed` 将前端构建产物打包进二进制 |
